# 专业课知识库快速工作流

这套流程的目标是：换一门专业课时，不改 Python 代码，也能快速建立一个可核对、可恢复、不会重复入库的私有知识库。

## 第一次使用

1. 运行 `启动知识库.bat`，或执行：

   ```powershell
   python -m streamlit run app_kb.py --server.port 8501 --server.fileWatcherType none
   ```

2. 打开“1 导入并核对文本”。
3. 如果列表里没有你的专业课，展开“＋ 新建一门专业课知识库”。只填专业课名称即可；考试代码、本地资料文件夹和抽取重点都可以留空。
4. 上传 PDF、图片、TXT，或直接粘贴文本。
5. 在“清洗后 / 可继续修正”中人工检查文字。确认无误后，点击“文本已核对，生成候选知识点”。
6. 在“2 确认知识点”逐条修改、删除或确认。
7. 只有名称、核心定义和原文依据都完整的确认项，才能保存进“3 我的知识库”。

## 中途退出怎么办

系统会把以下内容保存到 SQLite：

- 资料基本信息和处理状态
- 原始提取文本、清洗文本和人工确认文本
- PDF/OCR 处理报告和错误信息
- 剩余草稿、已确认队列和警告

重新打开页面后，使用顶部“继续上次未完成的资料”即可恢复。只想先修正文档、不立刻调用 AI 时，可以点击“保存文本，稍后继续”。

## 批量补资料

- 多文件上传时填写一个批次主题，系统会自动把文件名拼到章节名后。
- 每个文件的提取结果都会单独写入数据库，不会因为页面只打开最后一份而丢失前面的资料。
- 当前页面打开最后一份；其余未完成资料可从恢复入口继续。
- 同一批确认内容重复提交时，数据库去重键会跳过重复知识点，资料计数按真实关联行重新计算。

## 新增专业课后保存在哪里

- 内置学科配置：`professional_knowledge/default_subjects.json`
- 页面向导创建的个人配置：`data/config/custom_subjects.json`
- 私有知识库和工作流状态：`.env` 中 `MEMORY_DB` 指向的 SQLite 文件
- 用户上传文件：`data/user_materials/<user_id>/`

个人配置、数据库和上传资料默认不提交到 Git。

当前 `custom_subjects.json` 对同一个部署实例全局生效；如果未来开放给多个互不信任的用户，需要再增加管理员权限和按用户隔离的学科配置表。

## 学科配置能控制什么

每门专业课使用同一份配置统一控制：

- 页面名称与启用状态
- 本地资料目录
- 每次最多抽取的知识点数 `max_points`
- 学科抽取重点 `extraction_guidance`

长资料的分块数超过知识点额度时，系统会均匀覆盖全文的首段、中段和末段，并明确提示本次没有逐段处理的数量。

## 当前边界

本轮优先保证“导入、人工核对、草稿确认、可靠入库、恢复继续”。以下能力留到后续迭代：

- source chunks 页级/题级事实库
- FTS5 与向量混合检索
- 可视化知识关系图
- 后台队列和并发任务调度
- MinerU 等复杂版面解析

在当前闭环稳定前，不建议先叠加这些组件。
